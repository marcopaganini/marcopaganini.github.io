Go and Github

This is a very brief guide on how to set up your work environment to create Go programs.

Go uses the concept of a “workspace” to organize your source code and binaries. A workspace is basically a directory with three sub-directories under it: src, pkg, and bin.This sructure is explained in detail in the excellent (How to Write Go Code) article.

Setup your github directory structure

I find it useful to download clone all my github repositories under a fixed directory structure. In this case, I’d recommend ~/github as the base directory. Every cloned project will go under ~/github/<repo-username>. To set this up, create the initial github directory:

  mkdir ~/github

As a test, let’s clone one particular project out of github. Suppose you want to clone my github “termschool” vim theme, located at https://github.com/marcopaganini/termschool-vim-theme. In this case, notice the github username (marcopaganini). To clone this project, create a directory called marcopaganini under ~/github and clone it there using git clone:

  mkdir ~/github/marcopaganini
  cd github/marcopaganini
  git clone git@github.com:marcopaganini/termschool-vim-theme.git
